1. 总体目标与技术选择
目标: 重构现有基于 MATLAB patternsearch+IMR 的拟合代码，提供一个带 GUI 的、可扩展的 fitting/simulation 平台。
语言建议:
主框架与 GUI 使用 Python（含优化算法、参数管理、文件 IO）。
IMR 求解可先用 Python 实现；如性能不足，再将核心 ODE/IMR 部分用 C++ 实现成库/可执行文件，由 Python 调用。
平台: Windows 优先，尽量保持代码本身跨平台（不强依赖 Windows-only API）。
2. 数据与单位
GUI 中所有文本、按钮、菜单使用 英文，即使需求文档是中文。
实验数据加载：
从通用 .mat 文件加载，要求包含两个等长 1D 变量（默认名为 R、t），单位默认是 米/秒。
尽量支持关键字匹配（r, radius, R1，以及 time、t 等）自动识别列名。
将来如需要，用户可以在 GUI 中指定/更改单位。
不再依赖直接读取 ellipse_fitting_results.mat 的旧流程（会重新规范输入）。
无量纲化/有量纲视图切换：
“Dimensional” 视图：R 用 μm、t 用 μs 显示。
“Normalized” 视图：只影响展示，不改变内部计算：
( t_{\text{nondim}} = (t - t_{R_{\max}})/tc )
( R_{\text{nondim}} = R / R_{\max} )
3. GUI 总体布局与菜单
窗口整体分为左右两栏：
左侧：参数输入区（包括物理模型选择、材料参数、上下限、是否参与拟合等）。
右侧：上半为 R–t 预览区，下半为参数输出区。
顶部菜单栏包含四个菜单：File、Module、Physics、Settings。
3.1 File 菜单
Load experiment data:
打开文件对话框，选择 .mat 文件。
解析出 R、t（带关键字匹配），按“数据与单位”约定处理。
成功加载后：右侧预览图中用 黑色方形散点绘制实验数据，并添加图例 exp data。
Load parameters:
读取仅包含参数的文件（与 MATLAB 现有 struct_vars 格式兼容，见第 7 节）。
Save parameters:
保存当前左侧参数设置（包括初值、fitting 上下限、scale 信息），格式与现有 fun_fitting_initialization 结构兼容。
Save fitted parameters/results:
保存完整拟合结果，包括：实验与拟合曲线、最佳拟合参数等（见 7.2 小节，兼容 main_RT_optimization 第 201 行的保存格式）。
3.2 Module 菜单
有两个模块：Simulation 和 Fitting。

二者 GUI 布局基本一致，差别：

Simulation 模块：
按钮文字为 Simulate。
左侧参数区：
显示所有物理/材料参数。
显示上下限（用于参考），但不启用“是否参与拟合”和 scale 选项（灰掉）。
点击 Simulate 后，运行单次仿真，右侧预览区显示仿真曲线（颜色区别于实验数据）。
显示进度条 + ETA（估算剩余时间即可，不必绝对准确）。
Fitting 模块：
按钮文字为 Fit。
左侧参数区：
每个参数有：
勾选框：Fit?（是否参与拟合）。
数值输入框：初始值。
下限/上限输入框（与 MATLAB 当前 lb/ub 一致）。
单位选择（例如 Pa, kPa, MPa）。
scale 模式选择：linear 或 logarithm（拟合时才生效；在 Simulation 模块中灰色不可改）。
进入 Fitting 模块且数据已加载后：
在 R–t 预览图上叠加一块淡灰色半透明窗口，用来显示拟合时间范围：
左右边界可拖动，表示 t_fit_start 与 t_fit_end。
这两个标量用于：
决定 LSQ 误差计算仅使用该区间数据。
可用于缩短仿真 t_span，提高单次仿真速度（上层由你控制）。
Fitting 时需要按钮或控件支持：
Pause 按钮：暂停当前优化（不销毁线程池）。
Stop 按钮：中止当前优化，保留迄今为止的最佳结果。
在 Fitting 过程中：

右上图（预览区）实时展示当前“历史最好”拟合曲线以及实验数据。
右下参数输出区实时更新“历史最好”参数表和 LSQ。
3.3 Physics 菜单
Bubble dynamics:

提供至少两种可选模型：Rayleigh–Plesset (RP) 与 Keller–Miksis (KM)。
使用统一的 IMR 框架，根据选择在动力学方程中 if 切换 RP/KM。
Constitutive models（本构模型）：

在 Physics 菜单底部提供：
Add constitutive
Delete constitutive
真正的“本构模型选择”按钮不放在菜单里，而是放在左侧参数输入区顶部（选项卡下方），方便查看当前使用的本构。
本构模型插件机制：
每个本构模型由：
一个代码包（Python/C++ 模块），只负责材料本构（含粘弹性等），不直接做 IMR 主循环。
一份 JSON 配置，记录：
本构名字（显示在 GUI 中）。
需要的材料参数列表（名称、默认值、单位、是否默认参与拟合）。
该本构所需的内部状态变量说明（例如是否有 lambda_nv 网格）。
本构包接口（建议约定）：
输入：strain(r0), strain_rate(r0), state_in(r0)（内部变量，如 Maxwell 的 viscous stretch）、material_params。
输出：stress_integral S（或必要时 stress(r0) 分布）、state_out(r0)。
加载本构时，系统会检查 JSON 与代码包是否匹配、能否正常运行一个简单测试；通过后即可在 GUI 中选择。
选择不同本构模型后：

左侧参数区自动更新显示该本构需要的材料参数；同一套参数结构在 Simulation 和 Fitting 中共用（Fitting 额外有“是否参与拟合”和 scale）。
3.4 Settings 菜单
Parallelism / Threads：
设置优化时使用的 线程数 / 进程数。
优化使用持久的线程/进程池（不会像某些 MATLAB 实现一样用一会儿就关）。
Optimization algorithm：
提供算法选项：
Pattern Search（保持当前 MATLAB 选项行为）。
Differential Evolution。
CMA-ES。
其他（如 dual_annealing, basinhopping）用于全局搜索或跳出局部最小。
Pattern search options（分组显示）：
常用选项：
InitialMeshSize
MeshContractionFactor
MeshTolerance
StepTolerance
UseParallel
高级选项（放在“Advanced pattern search options”折叠面板中）：
Display
Algorithm（例如 classic）
UseCompletePoll
PollMethod（如 GSSPositiveBasis2N）
ScaleMesh
Cache、CacheTol 等
4. Fitting 行为与误差计算
LSQ 误差定义：
利用 t_fit_start–t_fit_end 范围内的实验和仿真曲线。
通过插值（类似 MATLAB 中 interp1 + makima）将仿真 R(t) 对齐实验时间点。
在 GUI 中可选择是否使用有量纲（R in μm）或无量纲（R/Rmax）来计算误差；默认与当前 MATLAB 行为一致（常用 μm）。
所有对 LSQ 的计算可调用公共函数（类似你已有的 fun_LSQErr_interpolation）。
失败/异常处理：
若某一次仿真失败（ODE 崩溃、NaN、插值失败等），或数据不足以算 LSQ：
返回 LSQErr = 1e10（统一的“不可行点”标记）。
优化继续，不终止。
这一约定适用于：
单个仿真试算。
单个拟合步中的任意异常。
5. 右侧输出区与结果导出
右侧上半：R–t 预览图
显示实验数据（黑色方形点）。
显示当前仿真结果（有色线）。
在 Fitting 中显示当前历史最好拟合曲线。
支持 Dimensional / Normalized 切换。
显示/拖动拟合时间范围灰色窗口。
右侧下半：参数输出区
实时显示：
当前历史最佳拟合参数列表（名称、单位、数值）。
当前最小 LSQ 值。
LSQ vs iteration 的实时曲线（可放在同一块区域，或单独一个小图）。
导出格式（与 main_RT_optimization 现有格式兼容）：
和下列字段一致或超集：

save(save_option.save_name, ...
     't_sim', 'R_sim', 'U_sim', 'P_sim', ...
     't_sim_nondim', 'R_sim_nondim', ...
     't_exp', 't_nondim_exp', 'R_exp', 'R_nondim_exp', ...
     'struct_best_fit');
其中 struct_best_fit 与现有 MATLAB 中的结构兼容。

6. 参数结构与文件格式
仅参数文件（Load/Save parameters）：
与 fun_fitting_initialization 产生的 struct_vars 格式兼容：
每个元素包含 name, value, lb, ub, scale, group。
用于：
作为 Simulation/Fitting 的参数模板。
作为 Fitting 的初始 guess 及上下限。
完整结果文件：
如上 5 节所述，包含：
实验 R–t 数据（有量纲和无量纲）。
仿真 R–t 数据（有量纲和无量纲）。
最佳拟合参数 struct_best_fit。
GUI 左侧参数区显示：
name（只读）。
value（可编辑）。
lb/ub（可编辑）。
scale（lin/log，可改）。
group（如 expansion1, expansion2, collapse，主要用于选择要拟合的组）。
7. 健壮性与用户交互
所有长时间操作（Simulation / Fitting）：
显示进度条与 ETA。
响应 Pause / Stop。
不中断 GUI 响应（使用后台线程/进程）。
Fitting 被 Stop 时：
保留当前历史最好结果（曲线 + 参数），允许用户保存。
错误情况（文件读取失败、字段缺失、本构包不匹配等）：
GUI 友好报错，不让程序崩溃。
对本构包，报错信息里指出是“JSON–代码包接口不匹配”还是“内部运算异常”。
